54 // Configures a LLMQ and its DKG
55 // See https://github.com/dashpay/dips/blob/master/dip-0006.md for more details
56 struct LLMQParams {
57  LLMQType type;
58 
59  // not consensus critical, only used in logging, RPC and UI
60  std::string name;
61 
62  // the size of the quorum, e.g. 50 or 400
63  int size;
64 
65  // The minimum number of valid members after the DKK. If less members are determined valid, no commitment can be
66  // created. Should be higher then the threshold to allow some room for failing nodes, otherwise quorum might end up
67  // not being able to ever created a recovered signature if more nodes fail after the DKG
68  int minSize;
69 
70  // The threshold required to recover a final signature. Should be at least 50%+1 of the quorum size. This value
71  // also controls the size of the public key verification vector and has a large influence on the performance of
72  // recovery. It also influences the amount of minimum messages that need to be exchanged for a single signing session.
73  // This value has the most influence on the security of the quorum. The number of total malicious masternodes
74  // required to negatively influence signing sessions highly correlates to the threshold percentage.
75  int threshold;
76 
77  // The interval in number blocks for DKGs and the creation of LLMQs. If set to 24 for example, a DKG will start
78  // every 24 blocks, which is approximately once every hour.
79  int dkgInterval;
80 
81  // The number of blocks per phase in a DKG session. There are 6 phases plus the mining phase that need to be processed
82  // per DKG. Set this value to a number of blocks so that each phase has enough time to propagate all required
83  // messages to all members before the next phase starts. If blocks are produced too fast, whole DKG sessions will
84  // fail.
85  int dkgPhaseBlocks;
86 
87  // The starting block inside the DKG interval for when mining of commitments starts. The value is inclusive.
88  // Starting from this block, the inclusion of (possibly null) commitments is enforced until the first non-null
89  // commitment is mined. The chosen value should be at least 5 * dkgPhaseBlocks so that it starts right after the
90  // finalization phase.
91  int dkgMiningWindowStart;
92 
93  // The ending block inside the DKG interval for when mining of commitments ends. The value is inclusive.
94  // Choose a value so that miners have enough time to receive the commitment and mine it. Also take into consideration
95  // that miners might omit real commitments and revert to always including null commitments. The mining window should
96  // be large enough so that other miners have a chance to produce a block containing a non-null commitment. The window
97  // should at the same time not be too large so that not too much space is wasted with null commitments in case a DKG
98  // session failed.
99  int dkgMiningWindowEnd;
100 };
