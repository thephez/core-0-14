213 // Require that user allocate at least 945MB for block & undo files (blk???.dat and rev???.dat)
214 // At 2MB per block, 288 blocks = 576MB.
215 // Add 15% for Undo data = 662MB
216 // Add 20% for Orphan block rate = 794MB
217 // We want the low water mark after pruning to be at least 794 MB and since we prune in
218 // full block file chunks, we need the high water mark which triggers the prune to be
219 // one 128MB block file + added 15% undo data = 147MB greater for a total of 941MB
220 // Setting the target to > than 945MB will make it likely we can respect the target.
221 static const uint64_t MIN_DISK_SPACE_FOR_BLOCK_FILES = 945 * 1024 * 1024;
